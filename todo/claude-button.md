# Claudeボタン 要件定義

## 概要
Claude CLIをワンクリックで起動できるボタンをデフォルト表示する

---

## 機能要件

### FR-1: ボタン表示
- [ ] ターミナル入力欄の直下（`>` プロンプトの下あたり）にボタンを配置
- [ ] 常に表示される（非表示オプションは後で検討）
- [ ] ボタンラベル: 「Claude」または Claudeアイコン

### FR-2: クリック動作
- [ ] ボタンクリックで `claude` コマンドを現在のターミナルで実行
- [ ] フォーカスしているターミナルペインで実行
- [ ] カレントディレクトリで起動

### FR-3: 状態表示（オプション）
- [ ] Claude実行中はボタンをdisabled or スタイル変更
- [ ] 実行中インジケーター表示

---

## 非機能要件

### NFR-1: UI/UX
- [ ] 目立ちすぎない控えめなデザイン
- [ ] ターミナルの操作を邪魔しない配置
- [ ] ホバー時にツールチップ表示（「claude コマンドを実行」）

### NFR-2: パフォーマンス
- [ ] ボタンクリックから実行開始まで100ms以内

---

## 技術設計

### 影響ファイル
| ファイル | 変更内容 |
|---------|---------|
| `src/components/XtermWrapper.tsx` | ボタンコンポーネント追加 |
| `src/index.css` | ボタンスタイル追加 |

### IPC
- 既存の `session:write` を使用
- 送信データ: `claude\n`（改行でEnter扱い）

### コンポーネント構成
```tsx
// XtermWrapper.tsx内に追加
<div className="claude-button-container">
  <button
    className="claude-button"
    onClick={() => writeToTerminal('claude\n')}
  >
    Claude
  </button>
</div>
```
